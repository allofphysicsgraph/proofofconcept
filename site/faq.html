Scope

What is the role of proofs in the Physics Derivation Graph?
The project documents the algebraic manipulations needed for Physics.

The Physics Derivation Graph does not include geometric arguments
The Physics Derivation Graph does not include textual explanations about why algebraic manipulations are being carried out.
The Physics Derivation Graph is similar to a Schuam's outline, except it covers all known Physics and documents the relations between various governing expressions.
There are no automatic mechanisms for generating new content.
There are no new algorithms.
The Physics Derivation Graph is about documenting existing knowledge
The Physics Derivation Graph requires no research.
The Physics Derivation Graph does include boundary conditions and initial conditions.

Blindly following mathematical manipulations of expressions does not necessarily lead to physically valid outcomes.

Expression index is Godel number

Audience

Who is the audience for the Physics Derivation Graph?
Students, teachers, researchers

How could the Physics Derivation Graph benefit students?
For physics students, this project could address questions like, "how much of what is known have I learned?" and "what else is there to learn?"
For math students, this project could address questions like, "when would I apply what I am learning?" and "what is this useful for?"

How could the Physics Derivation Graph benefit teachers?
For physics teachers, this project could address questions like, "how much of what is known have I learned?" and "what else is there to learn?"
For math students, this project could address questions like, "when would I apply what I am learning?" and "what is this useful for?"

How could the Physics Derivation Graph benefit researchers in Physics?
This project provides a very detailed explanation of how complex derivations are carried out. It also explicitly enumerates the assumptions necessary to perform derivations.

How could the Physics Derivation Graph be differentiated from crackpot work?
The goal (documenting all known Physics) sounds crackpot-ish. The approach doesn't disallow crackpot statements -- checking derivation steps using a computer algebra system doens't prevent physically nonsensical conclusions.
The purpose of the Physics Derivation Graph is to document known knowledge. No new knowledge is expected.

About the Physics Derivation Graph

What is the output of the Physics Derivation Graph?
The output can is typically a graph visualization or a PDF of the same content in a linear text format.

How can the Physics Derivation Graph be visualized?
The graph output can be visualized in a web browser, a static GraphViz output, and other formats.

How can the Physics Derivation Graph be checked?
A computer algebra system can be used to verify that each inference rule has been applied correctly to input and output expressions.

How is the content stored?
The expressions, symbols, and connections are each stored in XML format. This decision was based on a comparative analysis of other options, including MathML and Mathematica.

What subfields of Physics is this project capable of including?
Anything from Classical mechanics to Topological quantum field theory should be capable of being included.

Can the Physics Derivation Graph be used to prove Physics?
No. "Proof" is not a well-defined term in Physics.
From a Stack Exchange answer,
Mathematical proofs relate to exactly how a MODEL will behave. They don't have much to do with how the real world behaves. If the mathematics is carried out correctly, then one has "proven" how the model will behave.
The reason for experimentation, is to find out if the completely fictional MODEL that somebody simply made up, behaves in any way the same, as observations suggest the real world behaves. If it doesn't, that is not an indication of a mathematical error, it simply means the fictional model is not a good description of the real world, and must be changed.

Does this approach apply to other fields?
There is not clear application of the concepts used in the Physics Derivation Graph to other fields (ie chemistry, linguistics, economics, mathematics, etc).

Comparison of the Physics Derivation Graph with other projects

How is the Physics Derivation Graph different than Wikipedia?
Wikipedia is text-based with hyperlinks.
In contrast, the Physics Derivation Graph is effectively language-independent.

How is the Physics Derivation Graph different than the Principia_Mathematica?
The ambitious effort of Whitehead and Russell to prove the foundations of mathematics is not similar to this project. "Proof" doesn't apply in Physics. This project is about documenting existing knowledge in a format which can be parsed by computers.

How is the Physics Derivation Graph different than Equation Map?

How is the Physics Derivation Graph different than Symbolab?

How is the Physics Derivation Graph different than HyperPhysics? HyperPhysics is similar to Wikipediain that it is text based for edges.
HyperPhysics has what appears to be a graph (hand drawn), but is actually a multi-level tree -- a concept diagram. For example, content in the Classical Mechanics subpage does not link to any other subject.
The number of levels can vary. For example, the main page links to mechanics links to Collisions links to Momentum.

How is the Physics Derivation Graph different than Mathematica?
Mathematica is a commercial product, whereas the Physics Derivation Graph is open source.
Mathematica is a computer algebra system, whereas the Physics Derivation Graph is documentation of known mathematical physics relations.
