An inference rule is the atomic step of logic that relates two mathematical expressions. An inference rule is a subcategory of logical connectives. Inference rules include both rules of inference and rules of replacement.

"Whereas a rule of inference is always applied to a whole logical expression, a rule of replacement may be applied to only a particular segment." (source)

Inference rule are taught in math classes and get used in derivations. For example, consider the step "given expression 1, add 2 to both sides of expression 1, which yields expressions 2." The inference rule is of the form "add a value to both sides of the input expression to yield the output expression," where the value in the example is 2. More complex inference rules like "integrate both sides of the expression" are taught in Calculus, and "differentiate both sides of the expression with respect to a variable" is taught in a class on PDEs.

Enumerating all the inference rules is vital to the Physics Derivation Graph because inference rules govern how derivations are described. In addition to the normal topics in math, inference rules need to be capable of describing transformations in bra-ket notation and Einstein summation convention.

What is an atomic step?
There are many ways to accomplish the same task in a derivation. For example, "add X to both sides of an expression" is equivalent to "subtract Y from both sides of an expression" where X=-Y. Similarly, "divides both sides of the expression by Z" is equivalent to "multiply both sides of the expression by W" where Z=1/W. The choice of which inference rules are considered for use in the Physics Derivation Graph is motivated by multiple objectives: minimize the number of inference rules, enable concise derivations (fewer steps is better), make derivations understandable to humans (improve readability), and make derivations checkable by a computer algebra system (simpler is better). These objectives can yield conflict, so a subjective choice is made by the project owner.

Basis in logic for inference rules
The Substitution Property of Equality seems to be the basis for a couple of the inference rules:

  • subLHSofExprXintoExprY
  • subRHSofExprXintoExprY
  • subXforY
The Distributive property, Commutative property, Associative property are not included in the inference rules.

Inference rules are effectively functions which take inputs (expressions, feeds) and produce outputs (expressions). The graph visualization does not account for the fact the order matters. For example, the inference rule "subLHSofExprXintoExprY" is sensitive to which expression is treated as X and which is Y. Written in functional form,
LHS_exprOut, relation_exprOut, RHS_exprOut, = subLHSofExprXintoExprY(LHS_exprX, relation_exprX, RHS_exprX, LHS_exprY, relation_exprY, RHS_exprY)
